## Project Overview

An end-to-end **Enterprise Business Intelligence solution** that demonstrates the complete lifecycle of a real-world retail analytics project. Built to mirror production implementations at Fortune 500 retail companies.

**NovaMart Retail** is a simulated multi-channel retailer used as the business scenario for this project. All data is **synthetic**, generated by a deterministic Python script (`Python/generate_dataset.py`) to produce realistic enterprise-scale datasets with intentional data-quality defects for ETL demonstration.

---

## Business Scenario

NovaMart Retail operates through two sales channels:

- **Brick-and-Mortar Stores** вҖ” 120+ locations across 35 US states
- **E-commerce Platform** вҖ” Representing 40% of total revenue

This project delivers a centralized analytics platform providing insights into Sales, Product Performance, Customer Behavior, Inventory, Returns, Profitability, Employee Performance, Regional Trends, and Shipping Logistics.

## Data Source

All source data is **synthetic CSV**, generated by `Python/generate_dataset.py`.

| Profile | Orders | Order Details | Purpose |
|---------|--------|---------------|---------|
| `development` | 50,000 | ~200,000 | Fast local testing and iteration |
| `production` | 500,000 | 2,000,000 | Full-scale performance and optimization testing |

The generator uses deterministic random seeds for reproducibility and injects intentional data-quality defects (NULLs, duplicates, invalid dates, referential gaps) documented in `Documentation/Data_Generation_Spec.md`.

## Quick Start

### 1. Generate Development Dataset

```bash
cd Python
pip install -r requirements.txt
python generate_dataset.py --profile development
```

This produces ~50,000 orders in `Dataset/` in under 60 seconds.

### 2. Generate Production Dataset

```bash
python generate_dataset.py --profile production
```

This produces 500,000 orders and 2,000,000 order-detail rows (takes several minutes).

### 3. Run Data Quality Tests

```bash
cd ..
python -m pytest tests/test_data_quality.py -v
```
